Choose to build your Iris project in Debug or Test or Release or Release-protected mode.
Note: If you build in Debug mode, the landing screen in the generated app will be Volt MX's debugger screen, which your test automation scripts will most likely not be prepared to navigate away from. If you're building your project for automated testing using Jasmine test framework, you should build it in Test mode. For all other automation frameworks (Ex: TestNG), you will most likely want to build it in Release or Release-protected mode.
If you choose to build your Iris project in Release mode, It will not enable additional security and build the application in release unprotected mode.
The Release-protected mode enables additional security features that can protect your native built applications by detecting attacks like tampering, swizzling, debugging, jail breaking (iOS), rooting (Android), and information disclosure. The feature is generally enabled in the build UI mode at "Project Settings -> Native -> iPhone/iPad/Watch or Android -> Protected Mode" in Iris. You can refer more information about the protected mode in Iris User Guide.

Note: If you try to build DekstopWeb in Release-protected mode, it will be built in Release mode.
