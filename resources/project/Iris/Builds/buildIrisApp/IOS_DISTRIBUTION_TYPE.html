The purpose for which you will distribute this build of your app for iOS Native channels:
  • Development: If you're building your app for testing and/or debugging.
  • Ad hoc: If you're building your app for QA and/or UAT.
  • Enterprise: If you're building a release to distribute your app internally and your company is enrolled in Apple's Developer Enterprise Program. If so, remember to build in Release mode.
  • App Store: If you're building a release to distribute your app publicly through the Apple App Store. If so, remember to build in Release mode.
If you're building your app to test it using App Factory's DeviceFarm integration, this choice is not relevant and you may select any of the above options.
Note that both "Development" and "Ad hoc" will allow you to install and run on devices you physically control, so long as you have registered them in your Apple Developer Account, but only "Ad hoc" will allow you to test using Apple's production-level App Services such as Push Notifications and so on.
